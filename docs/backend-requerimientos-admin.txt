## 1. AUTENTICACIÓN — CRÍTICO (páginas reales: /sign-in, /recover-email, /reset-password, /email-review)
# Flujo: login → accessToken + refreshToken → almacenado en next-auth.
# Las rutas bajo /api/admin/ requieren token válido (401 → redirect /sign-in).

POST /api/admin/auth/login          # body: { email, password }
                                    # resp: { accessToken, accessTokenExpiresAt, refreshToken, refreshTokenExpiresAt, user }
POST /api/admin/auth/refresh        # body: { refreshToken }
                                    # resp: igual formato que login
POST /api/admin/auth/logout         # cierra sesión
POST /api/admin/auth/recover-email  # body: { email }
POST /api/admin/auth/reset-password # body: { password, token }
POST /api/admin/auth/recover-verify # body: { token } — usado por /email-review


## 2. LISTADOS PRINCIPALES — CRÍTICO (usados por hooks api.{entity}.list() en tablas)
# Estos endpoints son consumidos por las páginas de listado que usan useAgencias(),
# useFlota(), useRutas(), useViajes() — hooks que llaman api.{entity}.list().

GET /api/admin/agencias             # → Agencia[]  — usado por /agencias, drilldown
GET /api/admin/flota                # → Bus[]      — usado por /flota, drilldown
GET /api/admin/rutas                # → Ruta[]     — usado por /rutas, drilldown
GET /api/admin/viajes               # → Viaje[]    — usado por /viajes, drilldown

# Filtros comunes (query params): ?from=&to=&page=&limit=&agenciaId=&estado=
# Las tablas de listado muestran columnas que cruzan FK con nombres resueltos
# (ej: idAgencia → razónSocial, idTerminalOrigen → terminal.nombre).


## 3. REPORTES — CRÍTICO (páginas reales: /reportes-y-analitica/[slug])
# El frontend NO llama a /api/admin/reports, sino directamente al servidor de
# reportes externo vía serverHttpClient.

GET /api/reports/:slug              # → { report: ReportDefinition, rows: ReportRow[], meta }
                                    #   slug ∈ { ventas, viajes, manifiesto-sutran }
GET /api/reports/:slug/export       # → Excel binario (Content-Type: application/vnd.openxmlformats)
